Two changes:

1. **globals.css line 7** — Fix the dark variant selector. Change:
   ```css
   @custom-variant dark (&:is(.dark *));
   ```
   to:
   ```css
   @custom-variant dark (&:where(.dark, .dark *));
   ```
   This uses `:where()` for zero specificity and matches BOTH the `.dark` element itself AND its descendants. This is the official Tailwind v4 recommendation (per THEME-03, Pitfall 3 in RESEARCH.md).

2. **BaseLayout.astro** — Add a blocking inline script inside `<head>`, AFTER the `<meta name="generator">` line and BEFORE the closing `</head>` tag. The script must use `is:inline` to prevent Astro from bundling/deferring it:
   ```astro
   <script is:inline>
     document.documentElement.classList.toggle(
       "dark",
       localStorage.theme === "dark" ||
       (!("theme" in localStorage) && window.matchMedia("(prefers-color-scheme: dark)").matches)
     );
   </script>
   ```

   This script runs synchronously before the browser paints, reading localStorage first, falling back to system preference. It sets `.dark` on `<html>` so all CSS variables and Tailwind dark: utilities apply immediately (per THEME-03).

   Do NOT use a regular `<script>` tag — Astro will defer it and cause FOUC.

Create `src/components/layout/ThemeToggle.tsx` as a SolidJS component (per THEME-01, THEME-02):

```tsx
import { createSignal, createEffect } from "solid-js";
import { Moon, Sun } from "lucide-solid";

export default function ThemeToggle() {
  const [isDark, setIsDark] = createSignal(
    document.documentElement.classList.contains("dark")
  );

  createEffect(() => {
    document.documentElement.classList.toggle("dark", isDark());
    localStorage.setItem("theme", isDark() ? "dark" : "light");
  });

  return (
    <button
      onClick={() => setIsDark(d => !d)}
      aria-label={isDark() ? "Switch to light mode" : "Switch to dark mode"}
      class="w-9 h-9 flex items-center justify-center rounded-md hover:bg-muted transition-colors"
    >
      {isDark() ? <Sun size={18} /> : <Moon size={18} />}
    </button>
  );
}
```

Key implementation notes:
- Uses `client:only="solid-js"` when placed in Astro (done in Plan 02) — this component reads `document` at initialization time, which is not available during Astro's build-time SSR. Using `client:only` skips SSR entirely (per Pitfall 2 in RESEARCH.md).
- Signal initialized from `document.documentElement.classList.contains("dark")` — reads the state already set by the blocking inline script. Do NOT read from localStorage directly (per Anti-Patterns in RESEARCH.md).
- `createEffect` writes both DOM class and localStorage on every toggle (per THEME-02).
- Shows Sun icon in dark mode (switch to light), Moon icon in light mode (switch to dark).
- Import icons from `lucide-solid` NOT `lucide-react` (per Pitfall 6).
