<task type="auto">
  <name>Task 1: Build, measure bundle size, and deploy to Cloudflare Pages preview</name>
  <files>astro.config.mjs</files>
  <read_first>
    - package.json (confirm deploy:preview script exists from Plan 01)
    - wrangler.jsonc (confirm pages_build_output_dir: ./dist)
    - astro.config.mjs (current config — will temporarily add visualizer)
  </read_first>
  <action>
    **Step 1: Install bundle visualizer (dev dependency)**
    ```bash
    npm install rollup-plugin-visualizer --save-dev --legacy-peer-deps
    ```

    **Step 2: Temporarily add visualizer to astro.config.mjs**
    Add `import { visualizer } from 'rollup-plugin-visualizer';` at top.
    Add `visualizer({ emitFile: true, filename: 'stats.html' })` to the vite.plugins array after `tailwindcss()`.

    **Step 3: Build and measure**
    ```bash
    npm run build
    ```
    Then measure total JS size:
    ```bash
    ls -la dist/_astro/*.js
    ```
    Expected: ~22KB total (web.js 13KB + client.js 5.6KB + MobileMenu.js 2.2KB + ThemeToggle.js 1.4KB). This is far below Next.js baseline of 100-300KB.

    Record the exact total JS size for the summary.

    **Step 4: Remove visualizer from astro.config.mjs**
    Remove the `import { visualizer }` line and the `visualizer(...)` call from vite.plugins. The stats.html in dist/ will be overwritten on next build.

    **Step 5: Verify wrangler auth**
    ```bash
    npx wrangler whoami
    ```
    If not authenticated, the executor should note this requires `npx wrangler login` (human action — wrangler opens a browser OAuth flow).

    **Step 6: Create preview project (if it does not exist)**
    ```bash
    npx wrangler pages project create blog-astro-preview --production-branch main
    ```
    If the project already exists, this will error — that is fine, continue.

    **Step 7: Deploy**
    ```bash
    npx wrangler pages deploy dist/ --project-name blog-astro-preview
    ```
    Record the deployed URL (should be https://xxx.blog-astro-preview.pages.dev or similar).

    **Step 8: Rebuild without visualizer to get clean dist/**
    ```bash
    npm run build
    ```
  </action>
  <verify>

  </verify>

    - `npm run build` still succeeds after visualizer removal
  </acceptance_criteria>
