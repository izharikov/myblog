For each of the 8 MDX files listed, replace the `export const meta = { ... };` block (and the blank line after the closing `};`) with a YAML frontmatter block. The `{/* ----- */}` comment divider must remain immediately after the frontmatter closing `---`.

Conversion rule — transform this pattern:
```
export const meta = {
    title: "...",
    slug: "...",
    logo: "...",
    description: "...",
    tags: ["A", "B"],
    date: "DD.MM.YYYY",
};

{/* ----- */}
```

Into this pattern (YAML frontmatter):
```
---
title: "..."
slug: "..."
logo: "..."
description: "..."
tags: ["A", "B"]
date: "DD.MM.YYYY"
---

{/* ----- */}
```

Field mapping rules:
- `title`: copy value as-is, wrap in double quotes
- `slug`: copy value as-is, wrap in double quotes
- `logo`: copy value as-is, wrap in double quotes
- `description`: copy value as-is, wrap in double quotes
- `tags`: copy array notation as-is (e.g., `["XP", "MVC", "Razor", "Performance"]`)
- `date`: copy value as-is, wrap in double quotes

For posts 6, 7, 8: their titles use backtick template literals (e.g., `` title: `Fix "Resource parameter is required" error...` ``). Convert to double-quoted YAML strings. If the title contains double quotes, escape them with backslash: `"Fix \"Resource parameter is required\" error..."`. Alternatively, use single-quoted YAML if the value contains backslashes.
