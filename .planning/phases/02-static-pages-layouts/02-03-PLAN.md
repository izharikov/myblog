**2. Rewrite `src/pages/index.astro`** — Replace the existing placeholder with the home page:

```astro
---
import { getCollection } from 'astro:content';
import BaseLayout from '@/layouts/BaseLayout.astro';
import HeroSection from '@/components/home/HeroSection.astro';
import BlogGrid from '@/components/blog/BlogGrid.astro';
import { parseDate, formatDate } from '@/lib/date';
import { siteConfig } from '@/config/site';
import { personJsonLd } from '@/lib/json-ld';

const posts = await getCollection('blog');
const sorted = posts
  .sort((a, b) => parseDate(b.data.date).getTime() - parseDate(a.data.date).getTime())
  .slice(0, 3)
  .map(post => ({
    title: post.data.title,
    description: post.data.description,
    slug: post.id,
    logo: post.data.logo,
    dateDisplay: formatDate(parseDate(post.data.date)),
    tags: post.data.tags,
  }));

const jsonLd = personJsonLd();
---
<BaseLayout title={siteConfig.name} description={siteConfig.description}>
  <script type="application/ld+json" set:html={JSON.stringify(jsonLd)} />
  <HeroSection />
  <BlogGrid posts={sorted} title="Latest Posts" showLink={true} />
</BaseLayout>
```

The home page: (a) renders HeroSection, (b) fetches all blog posts, sorts by date descending, takes first 3, (c) passes them to BlogGrid with "See All" link visible, (d) includes Person JSON-LD.

**5. Create `src/pages/blogs/[slug].astro`:**

CRITICAL: The file MUST be named `[slug].astro` and `getStaticPaths` MUST return `params: { slug: post.id }` — the file basename determines the Astro.params key. Using `post.id` as the slug value because `generateId` in content.config.ts already strips the numeric prefix.

```astro
---
import { getCollection, render } from 'astro:content';
import BlogLayout from '@/layouts/BlogLayout.astro';

export async function getStaticPaths() {
  const posts = await getCollection('blog');
  return posts.map(post => ({
    params: { slug: post.id },
    props: { post },
  }));
}

const { post } = Astro.props;
const { Content } = await render(post);
---
<BlogLayout
  title={post.data.title}
  description={post.data.description}
  date={post.data.date}
  slug={post.id}
  logo={post.data.logo}
  tags={post.data.tags}
>
  <Content />
</BlogLayout>
```

This renders the MDX content inside BlogLayout, which handles JSON-LD, article header, date formatting, and tags.
