**Anchor click script** (same in every heading file):
```html
<script>
  document.querySelectorAll('[data-anchor-id]').forEach(link => {
    link.addEventListener('click', (e) => {
      e.preventDefault();
      const id = link.getAttribute('data-anchor-id');
      const url = `${window.location.origin}${window.location.pathname}#${id}`;
      void navigator.clipboard.writeText(url);
      window.location.hash = id!;
    });
  });
</script>
```

NOTE: Astro deduplicates identical `<script>` blocks across components on the same page. Having the same script in all 5 heading files will result in it running once.

**CodeBlock.astro** -- Wraps `<pre>` element emitted by Shiki. Structure:
- Frontmatter: `const { class: className, ...rest } = Astro.props;`
- Outer div: `class="relative mb-4 mt-6 rounded-lg overflow-hidden border group"`
- Copy button: `<button class="absolute right-3 top-3 p-1.5 rounded opacity-0 group-hover:opacity-100 transition-opacity bg-background/70 hover:bg-background border z-10" aria-label="Copy code" data-copy-button>`
  - Button contains two SVGs side by side, one for copy icon (visible by default) and one for check icon (hidden by default):
    - Copy icon: lucide Copy SVG (`class="size-4 data-[copied]:hidden"` -- paths: `rect width="14" height="14" x="8" y="8" rx="2" ry="2"` and `path d="M4 16c-1.1 0-2-.9-2-2V4c0-1.1.9-2 2-2h10c1.1 0 2 .9 2 2"`)
    - Check icon: lucide Check SVG (`class="size-4 hidden data-[copied]:block"` -- path: `M20 6 9 17l-5-5`)
    - Both SVGs: `viewBox="0 0 24 24"`, `stroke="currentColor"`, `stroke-width="2"`, `fill="none"`, `stroke-linecap="round"`, `stroke-linejoin="round"`
  - Use `data-copied` attribute on the button to toggle icons
- Pre element: `<pre class:list={["astro-code overflow-x-auto p-4 text-sm", className]} {...rest}><slot /></pre>`

Copy script:
```html
<script>
  document.querySelectorAll('[data-copy-button]').forEach(btn => {
    btn.addEventListener('click', async () => {
      const wrapper = btn.closest('div');
      const code = wrapper?.querySelector('code');
      const text = code?.innerText ?? '';
      await navigator.clipboard.writeText(text);
      btn.setAttribute('data-copied', '');
      setTimeout(() => btn.removeAttribute('data-copied'), 2000);
    });
  });
</script>
```

For the icon toggle, use CSS attribute selectors in a `<style>` block within CodeBlock.astro:
```css
[data-copy-button] svg:last-child { display: none; }
[data-copy-button][data-copied] svg:first-child { display: none; }
[data-copy-button][data-copied] svg:last-child { display: block; }
```

**InlineCode.astro** -- Wraps inline `<code>` elements (no language class):
- Frontmatter: `const props = Astro.props;`
- Render: `<code class="relative rounded bg-muted text-primary py-[0.2rem] font-mono text-sm font-semibold px-1" {...props}><slot /></code>`

IMPORTANT: InlineCode will NOT be mapped to the `code` element in the components prop. Instead, handle inline code styling via global CSS in globals.css. Override only `pre` for code blocks. Reason: Shiki renders `<pre><code>...</code></pre>` -- overriding `code` would interfere with Shiki's output inside `<pre>`. Instead, add this to globals.css after the Shiki dark theme block:

```css
/* Inline code styling (backtick code outside of code blocks) */
:not(pre) > code {
  @apply relative rounded bg-muted text-primary py-[0.2rem] font-mono text-sm font-semibold px-1;
}
```

This means InlineCode.astro is created but will NOT be used in the components map. It exists as a standalone component if needed elsewhere. The inline code styling comes from global CSS.
