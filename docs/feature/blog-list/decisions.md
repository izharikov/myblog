# Blog List — Decisions

## 1. Layout approach

**Status:** Open

**Options:** A (featured + list), B (featured + 2-col), C (timeline), D (mixed density)

**Considerations:**
- A is the most scannable but loses visual richness
- B balances imagery and readability
- C is minimal/dev-blog aesthetic, very fast to scan
- D is the most visually dynamic but complex to maintain

## 2. Images on list page

**Status:** Open

**Options:** All cards have images | Only featured card | No images (list-only)

**Considerations:**
- Images add load time and visual noise for a tech blog
- A single featured image gives visual anchor without the wall-of-thumbnails effect
- No images is cleanest but may feel bare

## 3. Home page blog section

**Status:** Open

**Options:** Same component as /blogs | Dedicated compact variant | Show only 3 latest

**Considerations:**
- Home page has hero + blog section — blog section should be complementary, not dominant
- Showing 3-6 latest with a "See all" link is standard
